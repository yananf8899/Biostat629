# Project 2 — Longitudinal CEA Trajectories and CRC Disease Progression

Biostat 629, Winter 2026 — Yanan Fang.

This repository contains the full analytic pipeline, manuscript, slides, and
website for the Project 2 deliverable. The scientific question is whether
longitudinal CEA trajectory features (current value, 90-day slope, within-
patient variability) improve 180-day disease progression prediction in the
MSK-CHORD 2024 colorectal cancer cohort beyond standard clinical and genomic
covariates.

## Data access

MSK-CHORD 2024 is controlled-access. Obtain the release through cBioPortal
(<https://cbioportal.org>) and unpack it into a folder named `msk_chord_2024/`
at the project root. The R script expects the following files inside that
folder:

```
data_clinical_patient.txt
data_clinical_sample.txt
data_timeline_cea_labs.txt
data_timeline_progression.txt
data_timeline_diagnosis.txt
data_timeline_treatment.txt
data_mutations.txt
```

No data are committed to this repository.

## Reproducing the results

### 1. Software

- R 4.3.0 or newer
- R packages: `data.table`, `lme4`, `pROC`, `ggplot2`, `glmnet`
- TeX Live with `pdflatex` (for the manuscript)

The exact package versions used in the most recent run are written to
`sessionInfo.txt` at the end of the analysis.

### 2. One command for the analysis

```bash
Rscript project2_analysis.R
```

This generates `fig1_cea_trajectories.pdf` through `fig5_conformal_singletons.pdf`
plus the matching `.png` versions, and writes `sessionInfo.txt`. Random seeds
are fixed (`set.seed(629)` for the train/test split, the LASSO CV folds, and
the conformal calibration split), so every number cited in the manuscript
regenerates exactly.

### 3. Build the manuscript

```bash
pdflatex project2_report.tex
```


## Reproducibility notes

- `set.seed(629)` is set immediately before each random-split or CV-fold step.
- The script does not call `setwd()`, `rm()`, `source()`, or `system()`; the
  data path is a single relative variable (`data_dir <- "msk_chord_2024"`).
- The script does not modify the raw input files; all derived datasets live in
  R memory or are written to clearly-named output files.
- `sessionInfo.txt` is regenerated on every run and committed alongside the
  outputs to track environment drift.
- The full `landmark_time` is computed as days from CRC primary diagnosis
  (`t = 0` at diagnosis), so all time-varying covariates and the
  $(t, t+180]$ outcome window are on the same diagnosis-anchored scale.
